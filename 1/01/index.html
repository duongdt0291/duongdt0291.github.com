Constructor function, Class

Constructor function

Để khởi tạo 1 object, ngoài việc gán trực tiếp giá trị hoặc dùng Object.create, chúng ta có thể dùng constructor function.

Constructor function cũng như function thông thường khác trong JS. Tuy nhiên khi sử dụng cần thêm từ khóa new , và thường được viết hoa chữ cái đầu tiên để phân biệt với các function bình thường khác.

function User(name) {
	this.name = name
	this.greet = function() {
		console.log(`Hello, I'm ${this.name}`)
	}
}

const user1 = new User('duongdt')
user1.greet() // Hello, I'm duongdt

Ở bài trước, mình đã nói về thuộc tính [[Prototype]] ẩn của object. Vậy object được tạo ra bởi constructor function thì có prototype là gì, liệu có phải là Object.prototype không?

console.log(Object.getPrototypeOf(user1) === Object.prototype) // false

console.log(Object.getPrototypeOf(user1))
// { constructor: ƒ User(name)
// __proto__: Object }

console.log(user1.__proto__.__proto__ === Object.prototype) // true

Trong JS, mọi function F khi được tạo đều có thuộc tính prototype (thuộc tính nhé, phân biệt với khái niệm prototype của 1 object trong bài trước), trỏ đến 1 object có thuộc tính constructor chính là function đó.

Khi 1 object được tạo bởi function, object đó sẽ được gán [[Prototype]] = Fn.prototype.

Câu hỏi đặt ra là tại sao lại gán như vậy? Tại sao không gán thẳng cho thằng Object.prototype luôn mà phải tạo 1 thằng khác?

// VD trên
user1.hasOwnProperty('greet') // true

// Khai báo method greet cho User.prototype
function User(name) {
	this.name = name
}
User.prototype.greet = function() {
	console.log(`Hello, I'm ${this.name}`)
}

const user1 = new User('duongdt')
user1.greet() // Hello, I'm duongdt
user1.hasOwnProperty('greet') // false

Có thể thấy rằng, thay vì mỗi instance trực tiếp có phương thức greet , việc có 1 object chứa nó và các instance sẽ kế thừa từ object này sẽ có hiệu quả hơn về mặt bộ nhớ. Thay vì việc phải lưu 100 lần method greet cho 100 instance, lúc này chúng ta chỉ phải lưu 1 lần duy nhất ở object prototype.

Nói thêm về từ khóa new , nếu không dùng kèm từ khóa new thì function User bản chất chỉ là function bình thường, nhưng khi đi kèm new thì khi được gọi, những thứ sau sẽ xảy ra tự động

function User(name) {
	// Tạo 1 object kế thừa object User.prototype và gán cho biến this
	// const this = Object.create(User.prototype)
	
	// Code
	this.name = name

Inheritance

function Animal(name) {
	this.name = name
}
Animal.prototype.eat = function() {
	console.log(`${this.name} is eating`)
}

function Pet(name) {
	Animal.call(this, name)
}
Object.setPrototypeOf(Pet.prototype, Animal.prototype)

function Dog(name) {
	Pet.call(this, name)
}
Object.setPrototypeOf(Dog.prototype, Pet.prototype)

const milu = new Dog('Milu')
milu.eat()

Static method

Animal.isAnimal(obj) {
	return obj instanceof Animal
}

Class

Từ ES6, thay vì phải viết khá phức tạp như trên, chúng ta có thể viết bằng từ khóa class với cấu trúc đơn giản hơn nhiều.

class Animal {
	constructor(name) {
		this.name = name	
	}

	static isAnimal(obj) {
		return obj  instanceof Animal
	}

	eat() {
		console.log(`${this.name} is eating`)
	}
}

class Pet extends Animal{
	constructor(name) {
		super(name)
	}
}

class Dog extends Pet {
	constructor(name) {
		super(name)
	}
}

Khi dùng class , JS sẽ tự động thêm 1 thuộc tính ẩn cho class là [[FunctionKind]]: "classConstructor" . Dựa vào thuộc tính này:

  • JS sẽ yêu cầu bắt buộc sử dụng từ khóa new dùng class, thay vì ko bắt buộc như constructor function và 1 số điều khác.
  • Các method trong class sẽ được đặt enumerable bằng false.
  • Class luôn sử dụng strict mode.
